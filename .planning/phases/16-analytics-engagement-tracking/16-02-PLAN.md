<task type="auto">
  <name>Task 1: Create product engagement tracking utilities</name>
  <files>src/lib/analytics/product-engagement.ts</files>
  <action>
Create `src/lib/analytics/product-engagement.ts` with product-specific tracking helpers.

**Functions to implement:**

1. `useProductViewTracking(productSlug, productName)` - Hook to track time on product page
   - Records entry timestamp on mount
   - Tracks 'product_view_time' event on unmount with duration in seconds
   - Only tracks if duration > 3 seconds (meaningful engagement, not bounces)
   - Metadata: productSlug, productName, duration, deviceType

2. `trackFilterChange(filterType, value, resultCount)` - Track filter interactions
   - FilterType: 'category' | 'type' | 'search'
   - Value: selected filter value (e.g., 'women', 'perfume')
   - ResultCount: number of products after filter applied
   - Enables funnel analysis (which filter combinations lead to purchases)

3. `trackCategoryTransition(fromCategory, toCategory, method)` - Track category navigation
   - FromCategory: previous category slug (or 'shop' for main page)
   - ToCategory: new category slug
   - Method: 'click' | 'filter' | 'direct' (how user navigated)
   - Maps discovery flow patterns

4. `trackQuickView(productSlug, productName, duration)` - Track quick view interactions
   - ProductSlug: product identifier
   - ProductName: for human-readable reports
   - Duration: milliseconds modal/quick view was open
   - Distinguishes browsing behavior from deep product engagement

**Implementation notes:**
- Import `track` from '@vercel/analytics'
- Import `getDeviceType` from '@/lib/analytics/engagement-tracker'
- Use try/catch around all track() calls
- useProductViewTracking uses useEffect with cleanup
- Add TypeScript types for all parameters
